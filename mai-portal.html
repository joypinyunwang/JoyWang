MAI Portal

Description

Redesigning BodyMap Analytics

Project Type

Launched service; Worked as full-time Visual & UI designer at MAI

Team

3 designers, 1 frontend developer, 1 web developer

My role

I was responsible for flow iterations, standardized components for scalability through design systems, and handled UI design, prototyping, and communication with the frontend developer.

Duration

4 months (Oct 2022 - Jan 2023)

Tools

Figma

Overview

BodyMap Analytics is our existing website, providing administrators and instructors with an accessible web portal to assess learning progress, enhance curriculum design, and streamline exam preparation.

Our focus was on integrating VR classroom management, implementing personalized subscriber management, and optimizing the overall user experience.

Final Outcome

This product is exclusively available to company internal personnel, resellers, and other authorized users.

View here: https://app.mai.ai/

Problem Space

Understanding

I engaged with the UX designer and product manager to understand current user types, page flows for each function, and the essential MVP features for the product.

Key challenges

1. Complex user types & flows
2. Absence of design files

BodyMap Analytics Previous Version

Prompt

How can we make it easier for users to navigate personal subscriber and VR classroom management?

How do we create an efficient design system for collaborative work among multiple designers, ensuring easy access for front-end engineers?
